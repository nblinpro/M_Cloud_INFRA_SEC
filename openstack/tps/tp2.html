Implémentation de réseaux privés et flottants.

Flavors

Définition de gabarits de ressources sur-mesure.

1. Découverte et vérification

Nova est le cœur du calcul. Vérifions d'abord l'état des services et l'hyperviseur sous-jacent.

CLI - Compute Check
# Lister les services Nova opérationnels
openstack compute service list

# Vérifier l'hyperviseur actif (KVM/QEMU)
openstack hypervisor list

2. Création des Flavors

Les flavors définissent les ressources (vCPU, RAM, Disque) allouées aux instances.

Flavor Léger

1 vCPU | 512 MB RAM | 5 GB Disk

Flavor Standard

2 vCPU | 4 GB RAM | 20 GB Disk

Flavor IA

8 vCPU | 16 GB RAM | 50 GB Disk

CLI - Flavor Creation
openstack flavor create --vcpus 1 --ram 512 --disk 5 flavor_leger
openstack flavor create --vcpus 2 --ram 4096 --disk 20 flavor_standard
openstack flavor create --vcpus 8 --ram 16384 --disk 50 flavor_ia

3. Configuration Réseau

Mise en place de l'isolation réseau via Neutron pour les instances Nova.

CLI - Networking
# 1. Création du réseau
openstack network create private_tp2

# 2. Configuration du sous-réseau (192.168.100.0/24)
openstack subnet create --network private_tp2 \
  --subnet-range 192.168.100.0/24 \
  --gateway 192.168.100.1 \
  --dns-nameserver 8.8.8.8 \
  my-private-subnet

# 3. Création et configuration du routeur
openstack router create router_tp2
openstack router set router_tp2 --external-gateway public

# 4. Connexion du sous-réseau au routeur
openstack router add subnet router_tp2 my-private-subnet

4. Accès & Sécurité

Clés SSH

# Génération locale
                        ssh-keygen -f my-key
                        # Import dans OpenStack
                        openstack keypair create --public-key my-key.pub my-key

Sécurité (Firewall)

# Création du groupe
openstack security group create my-secgroup
                        # Règles (Ping, SSH, HTTP)
openstack security group rule create --proto icmp my-secgroup
openstack security group rule create --proto tcp --dst-port 22 my-secgroup
openstack security group rule create --proto tcp --dst-port 80 my-secgroup

5. Déploiement & Connectivité

Lancement de l'instance et allocation d'une IP flottante pour l'accès public.

CLI - Instance Launch
# Création de la VM vm-test1
openstack server create --flavor flavor_standard \
  --image cirros-0.6.3-x86_64-disk \
  --nic net-id=$(openstack network show private_tp2 -f value -c id) \
  --security-group my-secgroup \
  --key-name my-key \
  vm-test1

# Association de l'IP publique
openstack server add floating ip vm-test1 [IP_FLOATING]

Vérification : Ping réussi vers 8.8.8.8 depuis l'instance via le routeur NAT.

6. Snapshot & Monitoring

Sauvegarde de l'état de l'instance et surveillance des ressources.

CLI - Ops
# Créer une image à partir de l'instance
openstack server image create --name snapshot_vmtest1 vm-test1

# Consulter les stats de l'hyperviseur
openstack hypervisor stats show
