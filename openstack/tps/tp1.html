Installation OpenStack via DevStack

Guide complet pour déployer une infrastructure IaaS sur Ubuntu 24.04. Maîtrisez les services essentiels : Keystone, Nova, Neutron, Glance, Cinder et Swift.

Télécharger le sujet complet

Objectifs & Architecture

Objectif Principal

Installer une plateforme OpenStack complète (IaaS) sur Ubuntu 24.04 via l'outil officiel DevStack.

Services Déployés

  • Keystone (Identité)
  • Nova (Compute)
  • Neutron (Réseau)
  • Cinder & Swift (Stockage)

Préparation du Système

OpenStack ne doit jamais être exécuté en root. Nous créons un utilisateur dédié stack.

Bash
# 1. Mise à jour du système
sudo apt update && sudo apt upgrade -y

# 2. Création de l'utilisateur stack
sudo useradd -s /bin/bash -d /opt/stack -m stack

# 3. Droits sudo sans mot de passe
echo "stack ALL=(ALL) NOPASSWD: ALL" | sudo tee /etc/sudoers.d/stack

# 4. Connexion à l'utilisateur stack
sudo su - stack

Configuration DevStack

local.conf
[[local|localrc]]
ADMIN_PASSWORD=secret
DATABASE_PASSWORD=$ADMIN_PASSWORD
RABBIT_PASSWORD=$ADMIN_PASSWORD
SERVICE_PASSWORD=$ADMIN_PASSWORD

# Activation de Swift (Object Storage)
enable_service s-account s-container s-object s-proxy

Lancement du script d'installation :

Bash
./stack.sh

Accès Dashboard Horizon

URL par défaut : http://[IP_SERVEUR]/dashboard

Identifiants : admin / secret

Vérification des Services

Utilisation de la CLI OpenStack après chargement des variables d'environnement.

Bash
source openrc admin admin

1. Identité (Keystone)

CLI
openstack token issue

2. Compute (Nova)

CLI
openstack compute service list

3. Réseau (Neutron)

CLI
openstack network agent list

Compétences Validées

  • Déploiement automatisé d'un Cloud IaaS.
  • Compréhension de l'architecture modulaire d'OpenStack.
  • Administration avancée via CLI (Python Clients).

Installation & Vérification
OpenStack via DevStack

Guide complet pour déployer une infrastructure IaaS sur Ubuntu 24.04. Maîtrisez les services essentiels : Keystone, Nova, Neutron, Glance, Cinder et Swift.

Voir le sujet (PDF)

Objectifs & Architecture

Objectif Principal

Installer une plateforme OpenStack complète (IaaS) sur Ubuntu 24.04 via l'outil officiel DevStack.

Services Déployés

  • Keystone : Identité
  • Nova : Compute
  • Neutron : Réseau
  • Glance : Images
  • Cinder : Stockage Bloc
  • Swift : Stockage Objet

Pré-requis Matériels

Configuration Minimale

Pour test basique

  • RAM : 8 Go
  • CPU : 2 vCPU
  • Disk : 60 Go

Configuration Recommandée

Pour expérience optimale

  • RAM : 16–32 Go
  • CPU : 4–6 vCPU
  • Disk : 90+ Go

Config du TP (VMware)

  • OS : Ubuntu 24.04 Desktop
  • RAM : 32 Go
  • CPU : 6 Processeurs
  • Disk : 95 Go
  • Réseau : Mode NAT

1. Préparation du Système

Note de sécurité : OpenStack ne doit jamais être exécuté en root. Nous créons un utilisateur dédié stack.

# 1. Mise à jour du système
sudo apt update && sudo apt upgrade -y

# 2. Création de l'utilisateur stack (dossier home dans /opt/stack)
sudo useradd -s /bin/bash -d /opt/stack -m stack

# 3. Configuration des droits sudo sans mot de passe
echo "stack ALL=(ALL) NOPASSWD: ALL" | sudo tee /etc/sudoers.d/stack

# 4. Connexion à l'utilisateur stack
sudo su - stack

2. Installation de DevStack

A. Téléchargement

git clone https://opendev.org/openstack/devstack
cd devstack

B. Configuration (local.conf)

Création du fichier de configuration essentiel : nano local.conf

[[local|localrc]]
ADMIN_PASSWORD=secret
DATABASE_PASSWORD=$ADMIN_PASSWORD
RABBIT_PASSWORD=$ADMIN_PASSWORD
SERVICE_PASSWORD=$ADMIN_PASSWORD

# Activation de Swift (Object Storage)
enable_service s-account s-container s-object s-proxy

C. Lancement du déploiement

Durée estimée : 15 à 30 minutes.

./stack.sh

3. Accès & Dashboard Horizon

Installation Terminée

Une fois le script terminé, vous pouvez accéder au tableau de bord :

URL : http://192.168.80.128/dashboard (IP à adapter)
Utilisateur : admin
Mot de passe : secret

4. Configuration CLI & Vérification

Pour utiliser les commandes en terminal, il faut charger les variables d'environnement (authentification).

# Chargement des variables admin
source openrc admin admin

# Vérification du chargement
env | grep OS_

Tests des services & Résultats

# 1. Identité (Keystone)
openstack token issue
# 2. Compute (Nova)
openstack compute service list
# 3. Réseau (Neutron)
openstack network agent list
# 4. Images (Glance)
openstack image list
# 5. Stockage Bloc (Cinder)
openstack volume service list
# 6. Stockage Objet (Swift)
openstack object store account show

Gestion & Maintenance

Commandes Utiles

# Arrêter OpenStack proprement
./unstack.sh

# Nettoyer l'environnement (Reset total)
./clean.sh

# Redémarrer
./stack.sh

Vérification Apache

Horizon tourne sous Apache2. Vérifiez son état en cas de problème d'accès web.

sudo systemctl status apache2

Ce TP a permis de valider :

  • Compréhension de l'architecture modulaire OpenStack.
  • Administration via CLI et Dashboard Horizon.
