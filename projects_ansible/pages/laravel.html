Déploiement Automatisé Laravel via Ansible

Documentation technique du projet d'automatisation incluant la gestion de configuration, le déploiement sécurisé (Vault) et l'optimisation serveur.

1. Arborescence du projet

Structure du Repository
mon_projet_ansible/
├── hosts.ini                # Inventaire
├── site.yml                 # Playbook principal
├── group_vars/              # Variables partagées
│   ├── db/vault.yml
│   └── web/
│       ├── vault.yml
│       └── vars.yml
└── roles/
    ├── database/            # Tâches MySQL
    ├── php/                 # Install PHP 8.3
    ├── apache/              # Config HTTPD & VHost
    ├── composer/            # Install binaire
    ├── laravel/             # Déploiement App
    ├── selinux/             # Contextes de sécurité
    └── firewall/            # Ouverture des flux

2. Configuration Racine

Fichier d'inventaire (hosts.ini)

[web]
weblaravel ansible_host=192.168.80.167 ansible_user=innosys

[db]
bddlaravel ansible_host=192.168.80.168 ansible_user=innosys

Playbook Principal (site.yml)

---
- name: Déploiement Base de Données
  hosts: db
  become: yes
  roles:
    - database

- name: Déploiement Serveur Web
  hosts: web
  become: yes
  roles:
    - php
    - apache
    - composer
    - laravel
    - selinux
    - firewall

3. Variables & Sécurité (Vault)

Création des coffres-forts : sudo ansible-vault create group_vars/db/vault.yml

group_vars/db/vault.yml
mysql_root_password: "RootPass123!"
laravel_db: "laravel_db"
laravel_user: "laravel_user"
laravel_password: "LaravelPass123!"

Sécurisation des permissions (Hardening)

sudo chown -R nicolas:nicolas group_vars
sudo chmod 700 group_vars/db
sudo chmod 600 group_vars/db/vault.yml

4. Détails des Rôles

Role : Database (MySQL)

roles/database/tasks/main.yml
- name: Installer MySQL server et client
  dnf:
    name: [mysql-server, mysql, python3-PyMySQL]
    state: present

- name: Créer l'utilisateur Laravel (Restreint au serveur Web)
  mysql_user:
    name: "{{ laravel_user }}"
    password: "{{ laravel_password }}"
    host: "{{ web_server_ip }}"
    priv: "{{ laravel_db }}.*:ALL"
    state: present

Role : PHP (Remi Repo)

- name: Activer module PHP 8.3
  shell: |
    dnf module reset php -y
    dnf module enable php:remi-8.3 -y

Role : Sécurité (SELinux)

roles/selinux/tasks/main.yml
- name: Autoriser connexion réseau Apache
  command: setsebool -P httpd_can_network_connect on

- name: Appliquer contexte SELinux (Lecture/Ecriture)
  command: chcon -R -t httpd_sys_rw_content_t {{ item }}
  loop:
    - "{{ app_path }}/storage"
    - "{{ app_path }}/bootstrap/cache"

Synthèse du Projet

Cette infrastructure automatisée permet un déploiement idempotent. Chaque exécution garantit que le serveur est dans l'état souhaité, sans interruption de service.
