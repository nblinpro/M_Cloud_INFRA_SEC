Déploiement Samba via Ansible

Automatisation d'un serveur de fichiers sécurisé avec gestion des utilisateurs, des groupes et des permissions SetGID.

1. Arborescence du projet

Structure
mon_projet_ansible/
├── group_vars/
│   └── samba/
│       └── vault.yml
├── hosts.ini
├── roles/
│   └── samba/
│       ├── defaults/main.yml
│       ├── handlers/main.yml
│       └── tasks/main.yml
└── samba.yml

2. Configuration Racine

Fichier d'inventaire (hosts.ini)

INI
[samba]
srv_samba ansible_host=@IP_Machine ansible_user=user

Playbook Principal (samba.yml)

YAML
---
- name: Configuration du Serveur de Fichiers (Samba)
  hosts: samba
  become: yes
  vars_files:
    - group_vars/samba/vault.yml
  roles:
    - samba

3. Variables & Sécurité (Vault)

Coffre-fort Ansible Vault

Bash
ansible-vault create group_vars/samba/vault.yml
vault.yml
vault_samba_pass_test: "test"
vault_samba_pass_admin: "user"

Sécurisation des permissions

vault.yml
sudo chown -R nicolas:nicolas group_vars
sudo chmod 700 group_vars/samba
sudo chmod 600 group_vars/samba/vault.yml

4. Role : Samba

Variables par défaut (defaults/main.yml)

Ansible Tasks
---
samba_share_path: "/srv/partage_commun"
samba_group_name: "partageurs"
samba_user_test: "test"
samba_admin_user: "user"

Gestionnaire de service (handlers/main.yml)

Ansible Tasks
---
- name: Restart Samba
  ansible.builtin.service:
    name: smbd
    state: restarted

Tâches principales (tasks/main.yml)

Ansible Tasks
---
- name: Installer Samba
  ansible.builtin.apt:
    name: samba
    state: present
    update_cache: yes

- name: Créer le groupe de partage "{{ samba_group_name }}"
  ansible.builtin.group:
    name: "{{ samba_group_name }}"
    state: present

- name: Créer l'utilisateur test (sans login)
  ansible.builtin.user:
    name: "{{ samba_user_test }}"
    shell: /usr/sbin/nologin
    create_home: no
    groups: "{{ samba_group_name }}"
    append: yes
    state: present

- name: Ajouter l'admin "{{ samba_admin_user }}" au groupe "{{ samba_group_name }}"
  ansible.builtin.user:
    name: "{{ samba_admin_user }}"
    groups: "{{ samba_group_name }}"
    append: yes

- name: Configurer le dossier partagé (Permissions 2770)
  ansible.builtin.file:
    path: "{{ samba_share_path }}"
    state: directory
    owner: root
    group: "{{ samba_group_name }}"
    mode: '2770'

- name: Définir le mot de passe Samba pour test
  ansible.builtin.shell: >
    (echo '{{ vault_samba_pass_test }}'; echo '{{ vault_samba_pass_test }}') | smbpasswd -s -a {{ samba_user_test }}
  register: smb_test_result
  changed_when: "'Added user' in smb_test_result.stdout"
  no_log: true

- name: Définir le mot de passe Samba pour l'Admin
  ansible.builtin.shell: >
    (echo '{{ vault_samba_pass_admin }}'; echo '{{ vault_samba_pass_admin }}') | smbpasswd -s -a {{ samba_admin_user }}
  register: smb_admin_result
  changed_when: "'Added user' in smb_admin_result.stdout"
  no_log: true

- name: Configurer le partage dans /etc/samba/smb.conf
  ansible.builtin.blockinfile:
    path: /etc/samba/smb.conf
    marker: "# {mark} ANSIBLE MANAGED BLOCK - PARTAGE COMMUN"
    block: |
      [DossierCommun]
          comment = Partage pour test et moi
          path = {{ samba_share_path }}
          browsable = yes
          read only = no
          guest ok = no
          valid users = @{{ samba_group_name }}
          create mask = 0660
          directory mask = 0770
          force group = {{ samba_group_name }}
  notify: Restart Samba

- name: Afficher les informations de connexion
  ansible.builtin.debug:
    msg:
      - "✅ Installation terminée avec succès !"
      - "🖥️ Windows : \\\\{{ ansible_host }}\\DossierCommun"
      - "🐧 Linux : smb://{{ ansible_host }}/DossierCommun"

Conclusion

Ce projet démontre la mise en place d'un serveur de fichiers collaboratif sécurisé via Ansible Vault et l'utilisation de permissions avancées Linux (SetGID) pour garantir la cohérence des droits d'accès au sein du groupe.
