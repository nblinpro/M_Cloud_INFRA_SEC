Industrialisation du
Monitoring Système

Déploiement automatisé d'une stack Prometheus / Grafana via Ansible et Docker Compose pour la surveillance en temps réel des infrastructures Innosys.

01. Provisioning via Ansible

Avant de déployer les conteneurs, nous utilisons un Playbook Ansible pour préparer l'hôte (Localhost). Ce script automatise l'installation de Docker et des outils d'intégration VM.

install_base.yaml Ansible 
---
- name: Installation Docker et Open VM Tools
  hosts: localhost
  connection: local
  become: yes

  tasks:
    - name: Mettre à jour l'APT
      apt: { update_cache: yes }

    - name: Installer open-vm-tools
      apt: { name: open-vm-tools, state: present }

    - name: Ajouter le dépôt Docker et installer docker-ce
      apt_repository:
        repo: "deb [arch=amd64] https://download.docker.com/linux/ubuntu {{ ansible_distribution_release }} stable"
        state: present

    - name: S’assurer que Docker démarre
      service: { name: docker, enabled: yes, state: started }

02. Orchestration Docker Compose

La stack de monitoring est isolée dans un réseau dédié monitor. Elle comprend Prometheus pour la collecte, Grafana pour la visualisation, et cAdvisor/NodeExporter pour les métriques.

docker-compose.yml Docker 
version: "3.8"
services:
  prometheus:
    image: prom/prometheus
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
    ports: [ "9090:9090" ]

  grafana:
    image: grafana/grafana
    ports: [ "3000:3000" ]
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin

  cadvisor:
    image: gcr.io/cadvisor/cadvisor:latest
    volumes:
      - /:/rootfs:ro
      - /var/run:/var/run:ro

  nodeexporter:
    image: prom/node-exporter
    network_mode: host

03. Collecte des Métriques (Scraping)

Configuration du moteur Prometheus pour cibler les différents agents NodeExporter déployés sur le réseau Innosys (Intervalle : 5s).

prometheus.yml Prometheus 
global:
  scrape_interval: 5s

scrape_configs:
  - job_name: "nodeexporter"
    static_configs:
      - targets: ["192.168.80.152:9100", "192.168.80.151:9100"]

  - job_name: "apache-client"
    static_configs:
      - targets: ["192.168.80.151:9117"]

Résultat opérationnel

Grâce à cette approche IaC (Infrastructure as Code), le temps de déploiement d'une nouvelle sonde de monitoring a été réduit à quelques secondes, garantissant une observabilité totale des services critiques d'Innosys.
